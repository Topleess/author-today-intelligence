Как проверяется каждый вывод

  1. Факт: сырое значение, timestamp и URL источника.
  2. Расчёт: видимая формула, например 990 524 − 990 495 = +29.
  3. Комментарий: тег → наблюдение → точная цитата → полный текст → ветка → профиль → оригинал.
  4. Архив: отдельная нерегулярная точка, никогда не непрерывная линия.
  5. Гипотеза: по умолчанию отсутствует; должна быть явно отделена от факта.

Комментарии и границы

Из-за правил Author.Today продукт не выпускает массовый или авторизованный сборщик комментариев. Реальная ограниченная выборка выбирается владельцем вручную в обычном браузере и импортируется локально. Пароли, cookies, OTP, Authorization-заголовки и browser profile отклоняются.

Проверено: реальная ветка из двух публичных комментариев, два профиля, исходные даты/URL, parent/thread, один точный human-confirmed тег с confidence 1.0. Содержимое не публикуется в репозитории.

Запуск и проверка

git clone https://github.com/Topleess/author-today-intelligence.git
cd author-today-intelligence
docker compose --profile sync run --rm public-sync
docker compose up -d controller
curl http://127.0.0.1:8787/api/health

Контракт API read-only: попытка POST /api/summary должна вернуть HTTP 405. Unit/contract tests и privacy scan входят в проверку репозитория.

Чего здесь нет

  • SaaS, billing и облачного хранения;
  • автоматизации приватного кабинета;
  • психологических досье комментаторов;
  • модуля защиты авторских прав;
  • заявлений о причинах роста без данных.
